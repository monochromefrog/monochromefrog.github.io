Hugo To The Rescue
February 14, 2018
  • Thoughts
  • Hugo

Earlier this year I decided to reactivate my site’s blog section. Turns out things are not going pretty well. First, I’ve clean-installed my computer and forgot to install my trusty website generator app, Mixture. Second, Mixture developer discontinued the app and took it down from the website. Well, I’m screwed.

After playing with several static site generators listed at StaticGen, I am now trying to get along with Hugo. Why I choose Hugo? The main reason is I want to avoid the same pitfall as I did with Mixture. I made a short list of criteria to meet for my next static site generator:

  • The project has to be in an active development cycle.
  • Well maintained.
  • Has large user base.

Although these criteria are not a guarantee for the project’s future, at least I can go into positive thinking mode and say “Someone will take over the project in case the development stalled in the future”. Hugo theme creation process and the language used to code the template are similar to Mixture, these are plus points other than meeting my criteria. Theme and template creation do have learning curves in data related parts. Hugo uses the standard HTML, CSS, and JavaScript for the front end, therefore it’s accessible to everyone with basic web coding knowledge.

This ‘Getting along with Hugo’ process is an enjoyable detour from my illustration projects. Now that I have completed my initial site rebuilding project, I can say that Hugo performance is very good and it overcame limitations I had with Mixture. I will stick to Hugo for now and would recommend it to anyone looking for a fast and flexible static site generator. Next goal for my site will be adding bilingual option other than fixing quirk site behavior.
